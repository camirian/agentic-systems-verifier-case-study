# Release Boundary

This case study is intentionally limited to public-safe explanatory material.

## Excluded From This Public Case Study

- standards PDFs and copied technical reference documents
- grant application material
- original backend, frontend, and evaluation source code
- cloud deployment configuration
- planning notes, build logs, and operating instructions
- provider-specific model configuration
- credentials or environment files

## Included

- public problem framing
- architecture pattern summary
- demo link
- lessons learned
- explicit statement of what is not released

## Rationale

Publishing a sanitized case study preserves the portfolio signal without redistributing documents, implementation details, or operational assumptions that require separate review.
